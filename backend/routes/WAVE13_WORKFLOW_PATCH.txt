Goal:
Only mark a section as "Linked Sections" when the final bundle contains
multiple DISTINCT CRNs joined by cross-list/corequisite/mixed logic.

Do NOT mark ordinary same-CRN multi-line Banner rows as linked.

Apply these edits inside backend/routes/workflow.js.

1) In mergeBundleRows(rows, bundleId), after building all_crns / linked_sections,
   add this classification block before returning the bundle:

const distinctCrns = Array.from(new Set((bundle.all_crns || []).filter(Boolean)));
const linkedKinds = new Set(
  (bundle.linked_sections || []).flatMap((s) => {
    const kinds = [];
    if (s.cross_list) kinds.push("cross_listed");
    if (s.corequisite_crn) kinds.push("corequisite_bundle");
    return kinds;
  })
);

let bundle_type = "single";
if (distinctCrns.length <= 1) {
  bundle_type = (bundle.linked_sections || []).length > 1 ? "single_crn_multiline" : "single";
} else if (linkedKinds.has("cross_listed") && linkedKinds.has("corequisite_bundle")) {
  bundle_type = "mixed_bundle";
} else if (linkedKinds.has("corequisite_bundle")) {
  bundle_type = "corequisite_bundle";
} else if (linkedKinds.has("cross_listed")) {
  bundle_type = "cross_listed";
} else {
  bundle_type = "multi_part_course";
}

bundle.bundle_type = bundle_type;
bundle.distinct_crn_count = distinctCrns.length;
bundle.is_true_linked_bundle = distinctCrns.length > 1 && (
  bundle_type === "cross_listed" ||
  bundle_type === "corequisite_bundle" ||
  bundle_type === "mixed_bundle"
);

2) In the /available-sections query, make sure bundle_type and raw_row are returned if present.
   If bundle_type is stored only in raw_row, frontend can still read it from raw_row.

3) In the schedule upload insert/update, store bundle metadata in raw_row:

JSON.stringify({
  ...(section.raw_row || {}),
  linked_sections: section.linked_sections || [],
  bundle_type: section.bundle_type || "single",
  distinct_crn_count: section.distinct_crn_count || 1,
  is_true_linked_bundle: Boolean(section.is_true_linked_bundle),
})

4) If your available-sections response is mapping rows before sending, preserve:
   - raw_row.bundle_type
   - raw_row.distinct_crn_count
   - raw_row.is_true_linked_bundle
